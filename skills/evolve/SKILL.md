# Evolve — Continuous Evolution

This is what makes Agent Swarm different. The swarm doesn't just finish — it evolves. Skills are extracted, lessons recorded, documentation updated, and the next cycle is defined.

## Procedure

### 1. Apply fixes from Guard
```
| Gap | Fix | File | Verified |
|-----|-----|------|----------|
```

### 2. Update documentation (only if behavior changed)
```
| Document | Change | Why |
|----------|--------|-----|
```

### 3. Extract skills
If this cycle revealed a reusable pattern:
```
## Skill Candidate
Name: [descriptive]
Principle: [one sentence]
When to apply: [contexts]
Category: [research/review/pm/general]
Source: swarm-cycle [date]
```

### 4. Lessons learned
```
**Worked:** [what went well]
**Didn't work:** [harder than expected]
**Next time:** [improvement]
```

### 5. Define next cycle
```
**Next mission:** [goal]
**Type:** feature / bugfix / research / review
**Priority:** P1 / P2 / P3
**Trigger:** [when to start]
```

### 6. Cycle Report
```
## Swarm Cycle: [type] — [title]
Scout: [goal, N tasks, N roles]
Build: [X/Y completed]
Guard: [tests, compliance, criteria]
Evolve: [fixes, docs, lessons, skills extracted]
Result: [achieved]
Next: [next cycle]
```
