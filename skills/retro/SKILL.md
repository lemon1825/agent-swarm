# Retro — Retrospective Analysis

Like a hive reviewing its foraging patterns to optimize future flights, this skill collects metrics, finds patterns, and extracts lessons for continuous improvement.

## When to use
- End of sprint or cycle
- After a significant incident
- Periodic team reflection

## Procedure

### 1. Collect metrics
Gather data from the period under review:
```
## Metrics Collection

| Metric | Value | Trend |
|--------|-------|-------|
| Tasks completed | [count] | [up/down/flat] |
| Tests added | [count] | |
| Bugs found | [count] | |
| Cycle time (avg) | [duration] | |
| Success rate | [percentage] | |
| Lines changed | [count] | |
```

### 2. Analyze patterns
```
## Pattern Analysis

**Successes:**
| Pattern | Frequency | Impact |
|---------|-----------|--------|
| [what worked] | [count] | HIGH/MED/LOW |

**Failures:**
| Pattern | Frequency | Root cause |
|---------|-----------|------------|
| [what failed] | [count] | [why] |

**Bottlenecks:**
| Bottleneck | Impact | Suggested fix |
|------------|--------|---------------|
| [blocker] | [time lost] | [fix] |
```

### 3. Extract lessons
```
## Lessons Learned

| # | Lesson | Category | Action |
|---|--------|----------|--------|
| 1 | [insight] | Process/Technical/Communication | [what to change] |
```

### 4. Generate action items
```
## Action Items

| # | Action | Owner | Priority | Due |
|---|--------|-------|----------|-----|
| 1 | [specific action] | [who] | P1/P2/P3 | [date] |
```

### 5. Retro Report
```
## Retro Report — [period]

Success rate: [percentage]
Velocity: [tasks/cycle]
Top success pattern: [pattern]
Top failure pattern: [pattern]
Key lesson: [one sentence]
Action items: [count] ([P1 count] critical)
```

Output: Retro report with success rate, failure patterns, and improvements.
